# SPEC-UI-001 인수 기준

> Traceability: SPEC-UI-001 (spec.md §3) · 기획서 v0.2 §4.2 F5, §5, §8

## 1. 정상 시나리오 (Given-When-Then)

### AC-01. 대시보드 기본 렌더링 (REQ-004, 006)

- **Given**: items 25건, events 40건(new/price_drop/d7 혼합), 워치리스트 2개가 저장된 DB
- **When**: 대시보드를 실행하고 브라우저에서 접속한다
- **Then**: 물건 목록에 감정가·최저가(억/만 환산)·유찰 횟수·D-day가 표시되고, 이벤트 이력 타임라인이 렌더링되며, 면책 고지가 고정 표시되고 각 물건에 법원 원문 링크가 존재한다

### AC-02. 할인율 표시 (REQ-005)

- **Given**: enrich 결과 { discountRate: 0.32, sampleCount: 14, confidence: "높음" }가 있는 물건 A와 { discountRate: 0.15, sampleCount: 2, confidence: "낮음" }인 물건 B
- **When**: 대시보드를 렌더링한다
- **Then**: A는 "−32% (표본 14건 · 신뢰도 높음)"이 강조 표시되고, B는 "참고치 (표본 부족)"과 함께 강조 없이 표시된다

### AC-03. 필터 동작 (REQ-007)

- **Given**: 워치리스트 "인천 서구 아파트" 매칭 물건 7건 포함 전체 25건
- **When**: 워치리스트 필터로 "인천 서구 아파트"를 선택하고 이벤트 타입 `price_drop`을 선택한다
- **Then**: 조건에 해당하는 물건만 목록에 남는다

## 2. 엣지 케이스 시나리오

### AC-04. localhost 전용 바인딩 (REQ-002)

- **Given**: 대시보드 서버가 실행 중
- **When**: 127.0.0.1과 머신의 외부 인터페이스 IP로 각각 접속을 시도한다
- **Then**: 127.0.0.1 접속은 성공하고, 외부 인터페이스 IP 접속은 거부(connection refused)된다

### AC-05. 외부 네트워크 요청 0건 (REQ-003)

- **Given**: 네트워크 요청을 기록하는 테스트 환경(오프라인 모드)
- **When**: 대시보드 페이지를 로드하고 모든 화면을 탐색한다
- **Then**: localhost 이외의 네트워크 요청이 0건이다. 번들 산출물 정적 검사에서 외부 CDN URL이 발견되지 않는다

### AC-06. 읽기 전용 보장 (REQ-001)

- **Given**: 대시보드 실행 전 DB 파일의 체크섬
- **When**: 대시보드로 전체 화면 조회·필터 조작을 수행한 후 서버를 종료한다
- **Then**: DB 내용이 변경되지 않았다(체크섬 동일 또는 데이터 비교 일치). 대시보드 코드에 쓰기 쿼리 경로가 존재하지 않는다(정적 검사)

### AC-07. enrich 부재 시 정상 동작 (REQ-005 + SPEC-ENRICH-001 소프트 의존)

- **Given**: rt_trades가 비어 있고 enrich가 한 번도 실행되지 않은 DB
- **When**: 대시보드를 로드한다
- **Then**: 할인율 컬럼/라인 없이 나머지 화면이 오류 없이 렌더링된다

### AC-08. sync 실패/차단 배너 (REQ-008)

- **Given**: 마지막 sync_runs가 blocked=1이고 마지막 성공 sync가 2일 전인 DB
- **When**: 대시보드를 로드한다
- **Then**: 상단에 차단 경고 배너와 마지막 성공 sync 시각이 표시된다

### AC-09. 빈 DB 초기 상태

- **Given**: 마이그레이션만 적용된 빈 DB (첫 sync 이전)
- **When**: 대시보드를 로드한다
- **Then**: 오류 없이 "데이터 없음 — `auction-radar sync`를 먼저 실행하세요" 안내가 표시된다
