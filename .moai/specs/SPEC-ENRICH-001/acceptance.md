# SPEC-ENRICH-001 인수 기준

> Traceability: SPEC-ENRICH-001 (spec.md §3) · 기획서 v0.2 §6.6, §9, §12 (M3 DoD)

## 1. 정상 시나리오 (Given-When-Then)

### AC-01. 아파트 할인율 산출 — 신뢰도 높음 (REQ-004, 006, 008)

- **Given**: lawd_cd=28260(인천 서구), apt_name_norm="청라OO" 단지의 12개월 실거래 fixture 14건(전용면적 ±10% 내), 대상 물건 최저매각가 256,000,000원, 실거래 중위값 376,000,000원
- **When**: enrich를 실행한다
- **Then**: discountRate ≈ 0.319(−32%)가 **중위값 기준**으로 계산되고, confidence="높음", sampleCount=14, fallbackUsed=false로 반환된다

### AC-02. 캐시 갱신 제한 (REQ-002)

- **Given**: rt_trades에 lawd_cd=28260 × 202606 데이터가 이번 달 이미 fetch됨(fetched_at 최근)
- **When**: enrich 갱신을 다시 실행한다
- **Then**: 해당 조합에 대한 MOLIT API 호출이 발생하지 않고 캐시가 사용된다. 워치리스트에 없는 lawd_cd는 애초에 조회되지 않는다

### AC-03. 렌더링 계약 (REQ-011)

- **Given**: AC-01의 enrich 결과
- **When**: SPEC-ALERT-001 렌더러가 소비한다
- **Then**: "인근 실거래 중위값 대비 −32% (표본 14건 · 신뢰도 높음)" 형식 라인이 생성된다

## 2. 엣지 케이스 시나리오

### AC-04. 폴백 강등 (REQ-004, 008)

- **Given**: 단지명 매칭 후보 0건, 같은 lawd_cd + 면적 밴드 전체로는 12건이 있는 fixture
- **When**: enrich를 실행한다
- **Then**: 면적 밴드 폴백이 사용되고(fallbackUsed=true), 신뢰도가 "보통"(폴백 ≥ 10건)으로 강등된다

### AC-05. 신뢰도 등급 경계값 (REQ-008)

- **Given**: 동일 단지 표본이 각각 4건 / 5건, 폴백 표본이 각각 9건 / 10건인 4개 fixture
- **When**: 등급을 판정한다
- **Then**: 동일 단지 5건 → "높음", 4건 → "보통", 폴백 10건 → "보통", 폴백 9건 → "낮음"("참고치 (표본 부족)")

### AC-06. 표본 부족 강조 억제 (REQ-007, 008)

- **Given**: 표본 2건(신뢰도 "낮음")인 물건
- **When**: 알림 라인을 렌더링한다
- **Then**: 할인율이 굵게(`<b>`) 표시되지 않고 "참고치 (표본 부족)"가 병기된다. 신뢰도 병기 없는 할인율 단독 출력이 코드 경로상 불가능하다

### AC-07. 빌라·토지 참고치 고정 (REQ-009)

- **Given**: usage가 "다세대주택"으로 표준 카테고리 "빌라"에 매핑되는 물건
- **When**: enrich를 실행한다
- **Then**: 표본 수와 무관하게 신뢰도가 "참고치"로 고정되고 강조 표기가 억제된다

### AC-08. 쿼터 소진 / 키 부재 무중단 (REQ-003)

- **Given**: MOLIT mock이 쿼터 초과 오류를 반환하는 환경(또는 molitKey 미설정), 매칭 이벤트 3건 존재
- **When**: sync 파이프라인(수집→매칭→enrich→발송)을 실행한다
- **Then**: enrich는 skip되지만 알림 3건은 할인율 라인 없이 정상 발송되고, sync는 성공으로 종료된다. 기존 캐시가 있는 물건은 캐시 기준으로 할인율이 계산된다

### AC-09. lawd_cd 매핑 실패 (REQ-005)

- **Given**: lawd_cd가 NULL인 물건(3계층 매핑 실패)
- **When**: enrich를 실행한다
- **Then**: 해당 물건은 "실거래 비교 불가"로 기록되고 예외 없이 건너뛰며, 알림·매칭은 정상 동작한다

### AC-10. 중위값 outlier 방어 (REQ-006)

- **Given**: 실거래 표본 [3.5억, 3.6억, 3.7억, 3.8억, 0.9억(특수거래 outlier)] fixture
- **When**: 중위값을 계산한다
- **Then**: 결과가 3.6억(중위값)이며, 평균(약 3.1억)이 아님을 검증한다. 짝수 표본 케이스는 중앙 2값 평균 규칙을 따른다
