# fixtures 익명화 체크리스트 (REQ-018)

> 본 레포는 공개(PUBLIC) GitHub 저장소입니다. courtauction.go.kr 응답을 그대로 커밋하면
> 당사자 개인정보가 유출됩니다. 아래 체크리스트를 **모두 통과한 스냅샷만** `fixtures/` 에 커밋합니다.

## 원칙

- 커밋되는 fixture 는 파서·diff·정규화의 **구조 검증용**입니다. 실제 사건 데이터가 아니어야 합니다.
- 실서버 응답을 수집했다면, 반드시 익명화 스크립트/수작업으로 세탁한 뒤 커밋합니다.
- 의심스러우면 커밋하지 않습니다.

## 반드시 제거·치환할 항목

- [ ] **당사자 성명** (채권자/채무자/소유자/임차인 등 모든 실명) → `○○○` 또는 가공된 값
- [ ] **상세 주소의 지번·동호수** (예: "청라동 123-45 101동 1502호") → 읍면동까지만, 이하 마스킹
- [ ] **연락처·주민등록번호·사업자번호 등 식별번호** → 완전 삭제
- [ ] **실제 사건번호** → 가공된 사건번호(예: `2025타경100001`)로 치환
- [ ] **감정평가서/명세서 첨부 파일 경로·URL** → 삭제
- [ ] 응답 내 **세션 토큰·쿠키·CSRF 토큰** → 삭제

## 유지해도 되는 항목 (구조 검증에 필요)

- 법원 코드(B000280 등 공개 코드표 값)
- 용도(아파트/근린생활시설 등 카테고리 문자열)
- 금액(감정가/최저가) — 실제 사건과 무관한 가공 금액이면 유지
- 상태(진행중/유찰/변경/취하), 유찰 횟수, 매각기일(가공된 날짜)

## 커밋 전 확인 절차

1. `git diff --staged fixtures/` 로 추가되는 모든 값을 육안 검토
2. 성명/지번/식별번호 패턴 검색: 이름 2~4자 + 실주소 숫자 조합이 없는지 확인
3. 이 체크리스트의 모든 항목에 체크
4. 리뷰어 1인 이상이 익명화를 재확인
